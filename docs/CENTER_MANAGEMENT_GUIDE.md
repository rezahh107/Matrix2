# راهنمای سریع مدیریت مراکز

## افزودن مرکز جدید

### 1. ویرایش فایل Policy

```yaml
# policy.yaml
center_management:
  centers:
    # مراکز موجود...
    - id: 3  # شناسه جدید
      name: "مرکز جدید"
      default_manager: "نام مدیر پیش‌فرض"
      description: "توضیحات مرکز"
  
  priority_order: [1, 2, 3, 0]  # اضافه کردن به اولویت
```

### 2. بدون نیاز به تغییر کد!

- UI به طور خودکار مرکز جدید را نشان می‌دهد
- CLI از مرکز جدید پشتیبانی می‌کند
- الگوریتم تخصیص به طور خودکار مرکز جدید را پردازش می‌کند

## نمونه‌های استفاده

### نمونه ۱: دو مرکز ساده

```yaml
centers:
  - id: 1
    name: "گلستان"
    default_manager: "شهدخت کشاورز"
  - id: 2
    name: "صدرا" 
    default_manager: "آیناز هوشمند"
  - id: 0
    name: "مرکزی"
    default_manager: null

priority_order: [1, 2, 0]
```

### نمونه ۲: چندین مرکز با اولویت‌بندی پیچیده

```yaml
centers:
  - id: 1
    name: "مرکز الف"
    default_manager: "مدیر الف"
  - id: 2
    name: "مرکز ب"
    default_manager: "مدیر ب"
  - id: 3
    name: "مرکز ج"
    default_manager: "مدیر ج"
  - id: 0
    name: "سایر"
    default_manager: null

priority_order: [1, 3, 2, 0]  # اولویت خاص
```

## خطاهای رایج و راه حل

| خطا | دلیل | راه حل |
|-----|------|--------|
| `ستون 'manager_name' وجود ندارد` | فایل استخر ستون مدیران ندارد | ستون `manager_name` به فایل اضافه کنید |
| `هیچ مدیری برای مرکز X یافت نشد` | مدیر تعیین شده در استخر وجود ندارد | نام مدیر را تصحیح یا از مدیر دیگر استفاده کنید |
| `مرکز نامعتبر: Y` | مقدار مرکز دانش‌آموز نامعتبر است | دیتای دانش‌آموزان را بررسی یا از `default_center_for_invalid` استفاده کنید |
| `UI مراکز را نشان نمی‌دهد` | Policy بارگذاری نشده | فایل policy.yaml را بررسی کنید |

## بهترین روش‌ها

1. **همیشه مرکز 0 را تعریف کنید** - برای دانش‌آموزان بدون مرکز مشخص
2. **از شناسه‌های مثبت استفاده کنید** - به جز مرکز 0 که ویژه است
3. **مدیران پیش‌فرض تعیین کنید** - برای تجربه کاربری بهتر
4. **ستون مدرسه‌ای را تنظیم کنید** - اگر دانش‌آموزان مدرسه‌ای دارید
